La création des léxèmes: la fonction LexerNode

Les léxèmes sont créés grâce à une fonction globale nommée LexerNode.

var oLexeme = LexerNode( oScanResult )

Leur création influe la performance des lexers car elle prend du temps !
Plus l'objet est gros, plus le temps mis pour les créer est long.

Argument

Elle prend en argument l'objet oScanResult, issu de l'analyse lexicale.

Valeur retournée

Elle retourne un objet affecté avec l'attribut oValue : oLexeme.oValue===oScanResult.
L'objet retourné est soit :

  • Un élément HTML
    • Plus lent.
    • Permet d'afficher facilement le résultat de l'analyse.
    • Pour l'instant mon éditeur créé avec le Lexer Automaton est plus rapide avec (Il utilise pleinement les possibilités HTML !).

  • Un objet JavaScript implémentant l'interface des éléments HTML
    • Plus rapide (interface implémentée réduite).

Cet objet doit impérativement implémenter l'interface des éléments HTML.

Exemple de création de la fonction

var createLexerNode =function( sType ){
	var s = '{}'
	switch( sType ){
		case 'HTMLElement':
			s='document.createElement(o.token)';
			break;
		case 'ObjectNode':
			// Node est un objet utilisateur implémentant l'interface des éléments HTML
			s='new Node(o.token)';
			break;
		}
	return new Function ( 'o', 
		'var m = '+ s +';'
		+'m.oValue=o;'
		+'return m;'
		)
	}

var LexerNode = createLexerNode( 'HTMLElement' )
